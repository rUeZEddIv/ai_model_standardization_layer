PROJECT OVERVIEW
----------------
A production-ready NestJS RESTful API that provides a unified interface for 
multiple AI generation providers with standardized requests, responses, and 
webhooks across 9 generation categories.

ARCHITECTURE HIGHLIGHTS
-----------------------
1. Modular Design
   - Separate modules for each feature
   - Clean separation of concerns
   - Easy to extend and maintain

2. Provider Adapter Pattern
   - Abstract base adapter
   - Provider-specific implementations
   - Factory for dynamic selection

3. Field Mapping Strategy
   - Maps variable names only
   - Preserves original values
   - Follows API-specific requirements

4. Error Handling
   - Comprehensive try-catch blocks
   - Error logging
   - Graceful degradation
   - Automatic retry with failover

EXTENSIBILITY
-------------
Adding new provider:
1. Create adapter class extending BaseAdapter
2. Register in AdapterFactory
3. Add provider via API/seed
4. Add models for provider

Adding new category:
1. Create DTO in common/dtos
2. Add enum value in Prisma schema
3. Add endpoint in GenerationController
4. Implement adapter mappings
